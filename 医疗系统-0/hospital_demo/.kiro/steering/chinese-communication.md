# 中文交流规则

当用户使用中文与我交流时，我应该：

1. **使用中文回复** - 始终用中文回应用户的中文消息
2. **保持技术准确性** - 使用准确的中文技术术语，必要时可以在括号中标注英文原词
3. **代码注释中文化** - 在代码示例中使用中文注释来解释功能
4. **文档和说明中文化** - 创建的文档、README 文件等使用中文编写
5. **错误信息解释** - 用中文解释英文错误信息的含义

## 技术术语对照
- API - 应用程序接口 (API)
- Database - 数据库
- Controller - 控制器
- Service - 服务层
- Repository - 数据访问层
- Entity - 实体类
- DTO - 数据传输对象 (DTO)

## 代码风格
- 类名和方法名保持英文（遵循 Java 规范）
- 注释使用中文
- 变量名可以使用有意义的英文单词
- 文档字符串使用中文
